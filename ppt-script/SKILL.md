# 📘 教程讲稿转换器 (ppt-script)

## Overview

将用户提供的零散信息（**一句话、一个主题、一段教程等**），转化为结构清晰的教学内容。

系统按顺序生成以下两份独立材料（**必须先出大纲，后出讲解稿**），并**将它们保存到当前目录下按主题命名的文件夹（如 `YYYYMMDD_核心主题`）中**：

1. 📊 **PPT 教程大纲**（保存为 `YYYYMMDD_核心主题/大纲.md`）。给 `ppt-generator` 排版的脚手架。章节标题用乔布斯风表达核心观点，内容页用步骤化拆解，杜绝大段文字堆砌。
2. 🎬 **讲解口播稿**（保存为 `YYYYMMDD_核心主题/口播稿.md`）。口语化、贴近真实讲课状态，专业从容，逻辑清晰。

## 核心操作准则

### 一、 PPT大纲与口播稿的分工

1. **PPT 大纲**：只放关键词、短句、步骤要点。章节标题（titleSlide）用乔布斯风表达观点或处理思路。
2. **口播稿**：自然流畅地用"你/我/大家"拉近距离，专业、从容地讲解知识点，逻辑层层递进。

### 二、 教程式开场

开场需要**快速说明三件事**：

1. **今天讲什么**：一句话概括主题（如："今天聊一下怎么用 Claude Code 的 Skill 功能"）
2. **为什么要学**：简短说明学完能解决什么问题或获得什么能力
3. **大致流程预告**：让观众知道接下来会讲哪几个部分

不要平铺直叙地念背景，用简洁的设问或痛点引入即可。例如：

- "你有没有遇到过 XXX 的情况？今天我们就来彻底解决它。"
- "很多人觉得 XXX 很复杂，其实掌握这三步就够了。"

### 三、 章节标题与内容页的区别

1. **章节标题页（titleSlide）— 乔布斯风，紧贴主题**：

   - 必须是**一个完整的观点、结论或处理思路**，不是空洞的名词
   - 紧贴当前章节的核心内容，不用"第一章"、"第二节"等编号字眼
   - 用对比式、断言式、比喻式等手法，12字以内，一口气说完
   - 示例：
     - ✅ "AI不是代替你，而是放大你"
     - ✅ "三步搞定环境配置"
     - ✅ "别再手动改了，让AI帮你"
     - ❌ "第一章：AI概述"
     - ❌ "市场现状分析"
2. **内容页（listSlide、flowSlide 等）— 步骤化、条理清晰**：

   - 多用"第一步、第二步、第三步"的递进结构
   - 以**逻辑清晰、信息准确**为主
   - 合理拆解为步骤列表、要点清单、数据对比等

### 四、 视频实操/演示画面的穿插兼容

如果用户提供的素材包含"具体操作步骤、软件界面演示、代码实操"等需要**切换视频画面/共享屏幕**的内容：

1. **PPT 大纲**：单独提取为 `[动作指引: 实操演示穿插]`，该部分不生成 PPT 页面。
2. **口播稿**：在对应位置打上 `*(动作：此处切到屏幕录制/实操演示画面)*`。

## 规范输出流程

### Step 1: 理解主题与受众

根据用户提供的素材，分析：

- 目标受众是谁（初学者/有基础的人）
- 核心知识点有哪些
- 适合分几个章节讲解

### Step 2: 结构设计

将主题切分为**教程章节**：

- 开场引入（说明主题 + 学习目标 + 流程预告）
- 核心章节 × N（每个章节围绕一个知识点或操作步骤）
- 实操演示（如有）
- 总结回顾（要点梳理 + 下一步建议）

### Step 3: 严格文件落盘输出格式

**必须将两份文档写成本地文件，统一存放到专属目录下，以对接 `ppt-generator` 的工作流。**

获取当前系统日期（格式：YYYYMMDD），提炼核心标题。
在当前目录下创建 `YYYYMMDD_核心标题` 文件夹，写入以下两份文件：

#### 1. 写入大纲文件：`YYYYMMDD_核心标题/大纲.md`

*(该文件将作为调用 `ppt-generator` 时的直接输入)*

```markdown
# 📊 交付给 ppt-generator 的教程大纲

## [大纲结构: 封面]
- **主标题**: 用 Claude Code 的 Skill 打造你的专属工作流
- **副标题**: 从零开始，三步上手
- **系列**: AI 效率工具

## [大纲结构: 章节标题]
- **标题**: 什么是 Skill？一句话讲清楚
- **副标题**: 核心概念

## [大纲结构: 内容要点]
- **主题**: Skill 的三大核心组成
- **第一步**: 定义触发条件 — 什么时候自动启用
- **第二步**: 编写指令文件 — 告诉 AI 怎么做
- **第三步**: 绑定工具链 — 让 AI 有手有脚

*(注：此处切换演示镜头)*
## [动作指引: 实操演示穿插]
- **说明**: 讲师将在此处切出 PPT，演示从零创建一个 Skill 的完整过程。

## [大纲结构: 章节标题]
- **标题**: 别光看，动手才是硬道理
- **副标题**: 实战环节

## [大纲结构: 操作拆解]
- **主题**: 跟着做：创建你的第一个 Skill
- **第一步**: 新建 SKILL.md 文件
- **第二步**: 填写 name 和 description
- **第三步**: 编写核心指令
- **第四步**: 测试运行效果

## [大纲结构: 总结回顾]
- **要点回顾**: 今天学了什么
- **要点一**: Skill = 触发条件 + 指令 + 工具链
- **要点二**: 从简单场景开始，逐步迭代
- **下一步**: 下期我们聊聊如何优化 Skill 的触发精准度
```

#### 2. 写入口播稿文件：`YYYYMMDD_核心标题/口播稿.md`

```markdown
# 🎬 教程讲解稿 (时长预估: ~X分钟)

**【开场：主题引入】**
今天咱们来聊一个很实用的东西 — Claude Code 的 Skill 功能。
你有没有觉得，每次让 AI 帮你做事，都得重复说一大堆要求？
其实你完全可以把这些要求打包成一个 Skill，以后一句话就触发。
今天这期内容，我会带你从零开始，搞懂 Skill 是什么，怎么写，怎么用。

**【核心概念：什么是 Skill】**
*(动作：指向 PPT)*
简单来说，Skill 就是一份给 AI 的"说明书"。
你告诉它：什么情况下启动、启动之后按什么步骤做、可以用哪些工具。
搞清楚这三件事，你就掌握了 Skill 的核心。

**【操作拆解：三步搞定】**
我们来看具体怎么做。
第一步，定义触发条件……
第二步，编写指令文件……
第三步，绑定工具链……

*(动作：此处切到屏幕录制/实操演示画面)*

**【实战演示】**
好，概念讲完了，咱们直接上手。
我现在打开编辑器，从零创建一个 Skill……

**【总结回顾】**
好，今天的内容就到这里。
简单回顾一下重点：……
如果你也想试试，文件模板我会放在评论区。
下期我们聊怎么优化 Skill 的触发精准度，感兴趣的话别忘了关注。
```

### Step 4: 自评与后续操作

**在最终输出前，执行内部质量自查：**

1. **自我评分**：
   - 开场是否快速说明了"讲什么 + 为什么学 + 流程预告"？（0-25分）
   - 章节标题是否紧贴主题、乔布斯风、无"第几章"字眼？（0-25分）
   - 内容页是否步骤化、条理清晰？（0-25分）
   - 口播稿是否专业从容、逻辑递进、自然流畅？（0-25分）
2. **修正逻辑**：总分低于 80 分则重写。
