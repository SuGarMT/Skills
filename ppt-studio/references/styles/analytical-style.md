# 🔬 分析风 — 大纲写作规则

## 核心理念

用数据和证据说服人。标题是一个明确的结论或发现，内容用框架、对比和数据来支撑。适合行业分析、商业决策、技术选型、竞品对比等需要理性说服的场景。

## 章节标题规则

### 结论先行
标题直接是本章的核心发现或结论，让受众看到标题就知道「证据指向什么」。

- ✅ "成本下降 40% 的关键：自动化流水线"
- ✅ "方案 B 在三项指标上全面领先"
- ✅ "用户流失的 70% 发生在第三天"
- ✅ "React 仍是 2026 年的最优选"
- ❌ "成本分析"（太空泛，这是简洁风）
- ❌ "让我们看看成本怎么降的"（太口语，这是叙事风）

### 表达手法

| 手法 | 示例 |
|---|---|
| **数据结论** | "转化率提升 3 倍的底层逻辑" |
| **对比结论** | "A 方案 vs B 方案：性能差距 5 倍" |
| **归因分析** | "导致延期的三个根因" |
| **框架命名** | "SWOT 分析：我们的竞争位" |

### 禁止事项

- 不要用模糊的定性描述（"效果不错"→ 用"提升 27%"）
- 不要用感性表达和比喻
- 不要用祈使句
- 数据必须具体，不用"大约""大概"

## 内容页规则

- 善用对比结构（`comparison` 组件）：新 vs 旧、A vs B
- 数据页用 `stats` 组件，呈现 3 组核心指标
- 因果分析用 `flow` 组件展示逻辑链
- 多维度评估用 `card-grid` 组件
- 每个结论都要有支撑依据（数据/案例/引用）
- 图表优先于文字描述

## 示例大纲片段

```markdown
# 市场现状：三个不可忽视的信号 [title]
- subtitle: 2026 Q1 数据解读

# 核心指标概览 [stats]
- 月活用户: 1.2M (+35% YoY)
- 付费转化率: 4.7% (行业均值 2.3%)
- 客户留存率: 68% (90日)

# 用户流失的 70% 发生在第三天 [message]

# 流失原因归因 [card-grid]
- 新手引导缺失 → 42%
- 核心功能发现率低 → 28%
- 性能问题 → 18%
- 其他 → 12%

# 方案 A vs 方案 B [comparison]
- 方案 A: 优化新手引导 → 预期提升留存 15%，实施周期 2 周
- 方案 B: 重构核心体验 → 预期提升留存 30%，实施周期 6 周

# 建议：分阶段推进 [flow]
- Phase 1 (2周): 先做方案 A，快速止血
- Phase 2 (6周): 同步推进方案 B
- Phase 3 (持续): 数据监控 + 迭代优化
```
