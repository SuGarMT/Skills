# ⚡ 行动风 — 大纲写作规则

## 核心理念

让受众立刻动手。标题是一个明确的行动指令，内容是可执行的操作步骤。不讲为什么，只讲怎么做。适合实操教程、工具使用指南、速查手册。

## 章节标题规则

### 祈使句原则
标题用动词开头，告诉受众要做什么。

- ✅ "安装开发环境"
- ✅ "三步搞定数据库配置"
- ✅ "立刻上手：你的第一个 API"
- ✅ "跟着做：创建项目模板"
- ❌ "数据库配置介绍"（太被动）
- ❌ "关于数据库的那些事"（太文艺，这是叙事风）

### 表达手法

| 手法 | 示例 |
|---|---|
| **直接祈使** | "安装 Node.js" |
| **量化承诺** | "五分钟完成部署" |
| **跟着做** | "跟着做：写你的第一个测试" |
| **立刻开始** | "现在打开终端" |

### 禁止事项

- 不要用问句（那是叙事风）
- 不要用比喻和金句（那是乔布斯风）
- 不要有太多理论解释
- 每步操作必须可执行

## 内容页规则

- 每页以「操作步骤」为核心结构
- 步骤必须具体到可执行（不能是"配置好数据库"，要是"运行 `docker compose up -d`"）
- 代码/命令优先展示
- 文字说明控制在每步 1-2 句
- 可以标注预期结果："执行后你应该看到…"

## 示例大纲片段

```markdown
# 安装开发环境 [title]
- subtitle: 开始之前的准备

# 安装 Node.js 和 Git [flow]
- 下载 Node.js LTS 版本 → nodejs.org
- 安装 Git → git-scm.com
- 验证安装 → 终端运行 node -v 和 git --version

# 创建你的第一个项目 [terminal]
- mkdir my-project && cd my-project
- npm init -y
- npm install express

# 写第一个接口 [terminal]
- 创建 index.js
- 写入 Hello World 路由代码
- 运行 node index.js → 浏览器访问 localhost:3000

# 部署到线上 [flow]
- 注册 Vercel 账号
- 连接 GitHub 仓库
- 点击 Deploy → 获得线上 URL
```
