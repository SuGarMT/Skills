# 实现者子代理提示词模板

分派实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "Implement Task N: [task name]"
  prompt: |
    你正在实现任务 N：[任务名称]

    ## 任务描述

    [计划中任务的完整文本 - 粘贴在这里，不要让子代理读取文件]

    ## 上下文

    [场景设置：这个任务在哪里适用，依赖关系，架构上下文]

    ## 开始之前

    如果你对以下内容有疑问：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的地方

    **现在就问。** 在开始工作之前提出任何顾虑。

    ## 你的工作

    一旦你清楚了需求：
    1. 准确实现任务指定的内容
    2. 编写测试（如果任务要求则遵循 TDD）
    3. 验证实现是否有效
    4. 提交你的工作
    5. 自审查（见下文）
    6. 报告结果

    工作目录：[directory]

    **工作过程中：** 如果你遇到意外或不清楚的事情，**提问**。
    随时可以暂停并澄清。不要猜测或做假设。

    ## 报告之前：自审查

    用全新的眼光审查你的工作。问问自己：

    **完整性：**
    - 我是否完全实现了规格中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我没有处理的边界情况？

    **质量：**
    - 这是我最好的工作吗？
    - 命名是否清晰准确（匹配事物的功能，而非工作方式）？
    - 代码是否整洁且可维护？

    **纪律性：**
    - 我是否避免了过度构建（YAGNI）？
    - 我是否只构建了被请求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真正验证了行为（而非只是模拟行为）？
    - 如果要求了 TDD，我是否遵循了？
    - 测试是否全面？

    如果在自审查中发现问题，在报告之前立即修复。

    ## 报告格式

    完成后，报告：
    - 你实现了什么
    - 你测试了什么以及测试结果
    - 更改的文件
    - 自审查发现（如有）
    - 任何问题或顾虑
```
