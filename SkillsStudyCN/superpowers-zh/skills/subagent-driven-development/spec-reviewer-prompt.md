# 规格符合性审查者提示词模板

分派规格符合性审查者子代理时使用此模板。

**目的：** 验证实现者构建的内容是否符合请求（不多不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你正在审查一个实现是否符合其规格。

    ## 请求的内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得异常快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们关于实现内容的说法
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **要做的：**
    - 阅读他们编写的实际代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但实际缺失的部分
    - 查找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了请求的所有内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某些功能有效但实际没有实现？

    **多余/不需要的工作：**
    - 他们是否构建了未被请求的东西？
    - 他们是否过度设计或添加了不必要的功能？
    - 他们是否添加了规格中没有的"锦上添花"功能？

    **误解：**
    - 他们是否对需求的理解与预期不同？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式错误？

    **通过阅读代码来验证，而非信任报告。**

    报告：
    - ✅ 符合规格（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附带 文件:行号 引用]
```
