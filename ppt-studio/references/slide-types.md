# 17 种 PPT 组件 HTML 结构参考

生成 PPT（阶段 4）时，读取此文件获取各组件的 HTML 结构模板。所有组件结构必须**完全遵循选定的主题模板文件中的 HTML 标记方式**。

## 使用方式

1. 读取用户选定的主题模板文件（如 `themes/ultimate-dark.html`）
2. 找到模板中对应组件的 `<section class="slide type-xxx">` 结构
3. 保留结构，替换内容为大纲中的实际文本

## 组件速查表

### 1. cover — 封面

固定为第一页。展示演示的主标题、系列名称和关键词标签。

```html
<section class="slide type-cover">
  <!-- 参照主题模板中的 type-cover 结构 -->
  <!-- 必填: name(主题), series(系列), tagline(标签) -->
</section>
```

### 2. title — 章节标题

用于章节过渡，突出一个核心观点。大标题 + 副标题，纯文字居中。

### 3. message — 核心观点

一句话金句、核心论断。超大字号居中，极简背景。适合情绪高点或关键转折。

### 4. question — 提问页

引发思考、制造悬念。大问号装饰 + 问题文字。适合章节开头引导思考。

### 5. split — 左右分栏

左侧放文字描述，右侧放数据/卡片/代码。适合「观点+证据」并列展示。

### 6. stats — 数据统计

三列等宽数字卡片，适合展示 3 组核心 KPI 或对比数据。每组包含数字 + 标签 + 说明。

### 7. card-grid — 网格卡片

2×2 网格布局，展示 4 个并列的特性、优势或对比项。每个卡片包含标题 + 描述。

### 8. terminal — 代码展示

终端/代码编辑器风格的代码框。适合展示代码片段、命令行操作、配置文件。

### 9. comparison — 左右对比

左右两栏对称结构。适合新旧方案对比、A/B 方案对比、优缺点对比。

### 10. flow — 流程图

线性步骤节点（通常 3-4 步）。适合展示工作流程、操作步骤、因果链。

### 11. timeline — 时间线

时间轴 + 节点描述。适合项目里程碑、产品发展历程、历史沿革。

### 12. quote — 引用

大引号装饰 + 引用文字 + 来源/作者。适合名人名言、用户反馈、核心引述。

### 13. image-text — 图文说明

图片 + 文字并排。适合截图解说、产品界面展示、流程图配文说明。

### 14. list — 要点列表

编号或符号列表。适合简单的文字要点平铺、检查清单、总结回顾。

### 15. team — 团队介绍

头像 + 姓名 + 角色。适合团队展示、特约嘉宾介绍。

### 16. gallery — 图片画廊

网格或并排图片。适合多图展示、案例集、产品截图集。

### 17. end — 结束页

固定为最后一页。致谢 + 联系方式 + 可选二维码/链接。

---

## 关键约束

1. **每页必须适配 100vh**——内容不能溢出视口，如果一页放不下，必须拆成多页
2. **封面固定第一页，结束页固定最后一页**
3. **组件结构必须完全参照主题模板的 HTML 标记方式**——不要自创 class 名或结构
4. **保留 `<div id="slides-wrapper">` 包裹层**——这是翻页引擎的核心
5. **保留模板的 `<style>` 和 `<script>`**——不要修改 CSS/JS 逻辑
