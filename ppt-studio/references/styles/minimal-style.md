# 📋 简洁风 — 大纲写作规则

## 核心理念

用最少的文字传递最多的信息。标题直述主题，内容用精炼的要点列表，零修辞装饰。适合信息密度高、受众专业度高的场景。

## 章节标题规则

### 直述原则
标题就是这一章讨论的事物名称或动作，不加修饰。

- ✅ "环境配置"
- ✅ "核心架构"
- ✅ "API 接口说明"
- ✅ "部署流程"
- ❌ "让我们来聊聊环境配置"（太啰嗦）
- ❌ "三分钟搞定环境"（这是行动风）

### 格式规范

- 标题使用名词短语或动名词短语
- 副标题可省略，或用一句话概括本章范围
- 不使用比喻、反问、感叹号

## 内容页规则

- 每个要点一行，不超过 20 字
- 用 `→` 或 `-` 分隔层级
- 数据直接列出，不加修饰语（如"高达 95%"改为"95%"）
- 术语不避讳，面向专业受众

## 示例大纲片段

```markdown
# 环境配置 [title]
- subtitle: 开发环境准备

# 依赖安装 [list]
- Node.js >= 18
- Python >= 3.10
- Git

# 项目结构 [card-grid]
- src/ → 源代码
- tests/ → 测试用例
- docs/ → 文档
- config/ → 配置文件

# API 接口 [terminal]
- GET /api/users → 用户列表
- POST /api/auth → 身份认证
- PUT /api/settings → 更新设置
```
